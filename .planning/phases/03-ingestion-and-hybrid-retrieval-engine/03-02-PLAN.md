# Plan 03-02: Create Qdrant collection, sparse/dense indexing, payload indexes, and idempotent upserts

## Scope

Implement the indexing primitives needed to create/update the approved Qdrant hybrid collection and upsert chunk-level points repeatedly without changing IDs.

## Work Completed

- Added `lib/retrieval/sparse.ts` with deterministic tokenization, stable hashed sparse indices, corpus document-frequency stats, and BM25-style weights.
- Added `lib/retrieval/embeddings.ts` with:
  - `GeminiEmbeddingProvider` using the Gemini OpenAI-compatible embeddings endpoint.
  - `DeterministicMockEmbeddingProvider` for offline dry-run and tests.
- Added `lib/retrieval/qdrant.ts` with:
  - Hybrid collection creation for `text_dense` and `text_sparse` named vectors.
  - Approved payload indexes.
  - Idempotent point conversion and batched upsert.
- Added `scripts/ingest-corpus.ts` with dry-run and real-upsert modes.
- Added package scripts `corpus:extract`, `corpus:ingest:dry-run`, and `corpus:ingest`.

## Dry-Run Evidence

`pnpm corpus:ingest:dry-run` built 2,747 chunks and 2,747 sparse vectors with deterministic 32-dimension mock dense vectors. It did not attempt cloud calls.
