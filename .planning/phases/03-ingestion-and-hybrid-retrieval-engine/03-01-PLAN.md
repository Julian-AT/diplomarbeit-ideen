# Plan 03-01: Implement corpus-derived extraction, chunking, metadata, and deterministic IDs

## Scope

Extract corpus text from the approved raw archive, normalize it into document records, and produce deterministic chunk payloads with citation provenance.

## Work Completed

- Added `scripts/extract-corpus-text.py` for local extraction:
  - PDFs via PyMuPDF.
  - DOCX/PPTX via ZIP/XML text extraction.
  - Videos/images skipped for v1 metadata-only handling.
- Added `lib/retrieval/types.ts` for corpus documents, chunks, payloads, sparse vectors, and retrieval filters/results.
- Added `lib/retrieval/ids.ts` for stable deterministic chunk/thesis identifiers.
- Added `lib/retrieval/chunking.ts` for page-aware primary document chunking and smaller supporting chunks.
- Added `lib/retrieval/corpus-io.ts` for JSONL IO.

## Dry-Run Evidence

`pnpm corpus:extract` processed the real corpus and produced 62 text-bearing documents, skipped 24 non-text files, and reported no extraction errors.
